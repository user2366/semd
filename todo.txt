1. Для каждого показателя необходим рассчет разницы от прошлого месяца.
    1.1 Добавить проверку наличия данных в базе данных
2. Необходимо переделать процесс upload данных. Ускорить процесс загрузки данных.
3. Необходимо переделать логику отображения данных. Сейчас при каждом открытии страницы производится рассчет данных. 
   Необходимо для каждого закрытого месяца делать рассчет данных и выводить уже готовый отчет.
    3.1 Добавить новую модель данных для хранения готового отчета за месяц по отделению, по врачу.
4. Необходимо реализовать процесс отображения загрузки данных для клиентской части.
    4.1 Данный функционал доступен только для администратора
    4.2 Изначально открывается пустая страница, зачем через AJAX делаем запрос к базе данных.
5. Необходимо переделать очистку данных. Данные необходимо удалять по месяцам, а не все. Так же можно проработать удаление только не зарегистрированных данных.
    5.1 Данный функционал доступен только для администратора
6. Необходимо разместить минимальную версию программы на сервер.
    6.1 Необходимо настроить docker-compose.yml для развертывания программы.
